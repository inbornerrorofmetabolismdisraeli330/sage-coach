## 核心概念：认知注入 (Cognitive Injection)

传统 AI 应用的做法：

```
用户 → 你的 App → API 调用 → LLM → 返回结果 → 你的 App → 用户
```

问题：你在 LLM 前面加了一层 wrapper，这层 wrapper 通常在**限制** LLM 的能力。

明灯的做法：

```
用户 → Claude Code → [Skill 认知注入] → Claude (完整能力) → 用户
```

**Skill 不是限制 AI，而是增强 AI。** 你给 Claude 注入了一套它原本没有的专业认知架构 — 导师传承、阶段模型、场景策略、安全边界。Claude 的推理能力、共情能力、多语言能力全部保留，同时获得了专业引导能力。

这就像给一个极聪明的人一套完整的方法论培训。人没变，但能力维度变了。

```
│     └────────┴────────┴────────┘       │
│              ▼ 融合                     │
│  Layer 2: Stage Model (阶段模型层)      │
│  ┌─────┐   ┌─────┐   ┌─────┐          │

│  └─────┘   └─────┘   └─────┘          │
│              ▼ 应用                     │
│  Layer 3: Case Matrix (场景策略层)      │
│  ┌────────┐ ┌────────┐ ┌────────┐     │
│  │场景 A  │ │场景 B  │ │场景 C  │     │
│  │识别信号│ │识别信号│ │识别信号│     │
│  │应对策略│ │应对策略│ │应对策略│     │
│  └────────┘ └────────┘ └────────┘     │
│                                         │
│  ⚠️ Safety Boundary (安全边界)          │
└─────────────────────────────────────────┘
```

| 层 | 内容 | 作用 |
|----|------|------|
| **Persona Matrix** | 6-10 位导师的方法论 + 语言风格 + 经典语录 | 让 AI 有"师承" |
| **Stage Model** | 4-5 个引导阶段，每阶段有目标/方法/转换条件 | 让对话有方向 |
| **Case Matrix** | 5-8 个真实场景的识别信号和应对策略 | 处理现实复杂性 |
| **Safety Boundary** | 危机识别 + 专业转介 + 能力边界 | 不做超出范围的事 |

## 高级用法

### 组合使用

明灯的 4 个 Coach 可以组合：

```bash
# 创业低谷：先处理情绪，再理清产品
/crisis → (稳定后) → /startup

# 职业转型中的深层探索：理清方向后探索内在动机
/career → (发现深层困惑) → /spiritual

# 创业就是职业转型：从大厂到创业的完整路径
/career → (确定创业方向) → /startup

# 修行者的入世：灵性视角看职业和事业
/spiritual → /career 或 /startup
```

## FAQ

**Q: 和 "Act as a coach" prompt 有什么区别？**

A: 一个是"穿上白大褂"，一个是"读了 7 年医学院"。Prompt 是角色扮演，Skill 是完整的认知架构 — 有师承、有阶段、有场景、有安全边界。

**Q: 为什么不做成 SaaS 收费？**

A: 因为这个项目的存在理由就是"智慧不应该有门槛"。而且 Skill 形态比 SaaS 更强大 — 你有完全控制权和隐私。

**Q: 支持英文吗？**

A: 支持。Skill 用中文编写，但 Claude 会根据你的对话语言自动调整。

**Q: 可以用在 ChatGPT / 其他 AI 上吗？**

A: 可以把 SKILL.md 内容作为 system prompt 用在任何 LLM 上。但在 Claude Code 里效果最好。

**Q: 我想加一个导师进去怎么办？**

A: Fork → 在对应 SKILL.md 的导师原型部分加入 → PR。格式：导师名 + 核心方法论 + 语言风格 + 经典语录。
